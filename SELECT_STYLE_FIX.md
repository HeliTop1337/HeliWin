# Исправление стиля выпадающих списков (Select)

## Проблема
Выпадающие списки (select элементы) имели серый фон вместо темного стиля, что не соответствовало общему дизайну приложения.

## Решение
Обновлены все select элементы в приложении с применением темного стиля:

### Стиль select элементов:
```css
bg-black/50 backdrop-blur-xl border border-white/10 text-white
focus:ring-2 focus:ring-purple-500 focus:border-transparent
appearance-none /* убирает стандартную стрелку */
bg-[url('...')] /* кастомная SVG стрелка */
bg-[length:1.5rem] bg-[right_0.5rem_center] bg-no-repeat pr-10
```

### Стиль option элементов:
```css
className="bg-[#1a1a1f] text-white"
style={{ colorScheme: 'dark' }}
```

## Обновленные файлы:

### Frontend страницы:
1. **frontend/pages/inventory.tsx**
   - Выпадающий список сортировки (Недавние, По цене, По названию, По редкости)

2. **frontend/pages/admin/cases.tsx**
   - Выбор предмета для добавления в кейс

3. **frontend/pages/admin/promo.tsx**
   - Выбор кейса для промокода
   - Выбор предмета для промокода

4. **frontend/pages/admin/items.tsx**
   - Выбор редкости предмета (Сталкерское, Ветеранское, Мастерское, Легендарное)

## Особенности реализации:

### Кастомная стрелка
Используется SVG стрелка вместо стандартной браузерной:
- Белый цвет стрелки
- Размер 1.5rem
- Позиция справа с отступом 0.5rem

### Темная цветовая схема
- `colorScheme: 'dark'` - заставляет браузер использовать темную тему для нативных элементов
- Темный фон для выпадающего меню: `#1a1a1f`
- Полупрозрачный фон для самого select: `black/50`

### Адаптивность
- Фокус с фиолетовым кольцом для соответствия общему стилю
- Плавные переходы при взаимодействии
- Поддержка всех браузеров

## Результат
Все выпадающие списки теперь имеют единый темный стиль, соответствующий общему дизайну HeliWin.
